完善基本命令 drop table

Parser 模块

当一条 SQL 请求到达 Server 端后，首先经过的是 Parser 模块。Parser 模块主要作用是根据定义的语法规则判断一条 SQL 语句的格式是否符合对应的语法结构，将一条用户写的 SQL 请求转换成可以在数据库内核中所使用的数据结构。

miniob/src/observer/net/sql_task_handler.cpp 文件中定义了一个函数 RC SqlTaskHandler::handle_sql(SQLStageEvent *sql_event)，这个函数的前身目前并未研究。函数内部会依次调用query_cache_stage_，parse_stage_，resolve_stage_，optimize_stage_，execute_stage_五个函数，用于处理 SQL 语句的五个主要生命节点。

对于 Parse 模块:

会调用 parse_stage_(class ParseStage)中的handle_request方法来处理该阶段。至于函数的具体实现先跳过，直接用官方文档的话：

一条 SQL 语句，无论写得多么复杂，本质上就是一层一层嵌套的树状结构。Parser 模块这一个部分主要做的就是根据用户所写的语法规则正确的识别，并将一条 SQL 语句转化为一个语法树的结构。但一条符合语法树规则的 SQL 不一定是一条“合格”的 SQL，数据库中还存在着许多约束，一条 SQL 还必须满足语义的约束，许多语法的约束以及虚视图的展开都是在 Resolver 模块中完成的。

Resolver 模块

Resolver 模块主要作用是对 Parser 模块所生成的符合语法规则的树状结构进行进一步的约束检查，还可能会提取表达式的属性。
在 Resolver 这一层会转化成一个自己的数据结构。在 OceanBase 中用的是 STMT（Statement），这是非常重要的一个类。

对于 Resolver 模块:

会调用 resolver_stage_(class ResolverStage)中的handle_request方法来处理该阶段。
函数内部会创建 STMT 类，用于后续的处理。
