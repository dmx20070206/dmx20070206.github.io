完善基本命令 drop table

当用户发出一条 SQL 请求时，类SqlTaskHandler会创建五个成员变量分别管理五大模块。

在成员函数RC SqlTaskHandler::handle_sql(SQLStageEvent *sql_event)中，会依次调用五大模块query_cache_stage_，parse_stage_，resolve_stage_，optimize_stage_，execute_stage_的管理器做相关的处理。

Parser 模块

当一条 SQL 请求到达 Server 端后，首先经过的是 Parser 模块。Parser 模块主要作用是根据定义的语法规则判断一条 SQL 语句的格式是否符合对应的语法结构，将一条用户写的 SQL 请求转换成语法树。

对于 Parse 模块:

会调用 parse_stage_(class ParseStage)中的handle_request方法来处理该阶段。官方文档给出的解释是： >一条 SQL 语句，无论写得多么复杂，本质上就是一层一层嵌套的树状结构。Parser 模块这一个部分主要做的就是根据用户所写的语法规则正确的识别，并将一条 SQL 语句转化为一个语法树的结构。但一条符合语法树规则的 SQL 不一定是一条“合格”的 SQL，数据库中还存在着许多约束，一条 SQL 还必须满足语义的约束，许多语法的约束以及虚视图的展开都是在 Resolver 模块中完成的。

Resolver 模块主要作用是对 Parser 模块所生成的符合语法规则的树状结构进行进一步的约束检查，还可能会提取表达式的属性。 在 Resolver 这一层会将 SQL 语句转化成一个自己的数据结构。在 OceanBase 中用的是 STMT（Statement），这是非常重要的一个类。

优化模块

第四个模块rc = optimize_stage_.handle_request(sql_event);主要用于命令的优化。优化模块有两个非常重要的类：逻辑算子LogicalOperator和物理算子PhysicalOperator。 逻辑算子描述当前执行计划要做什么，物理算子描述执行计划将如何执行。

处理优化模块的顶层类为OptimizeStage，类中包括逻辑计划生成器、物理计划生成器、逻辑重写器。

创建、重写、优化逻辑算子

进入函数RC OptimizeStage::handle_request(SQLStageEvent *sql_event)中，第一部分就是创建逻辑算子 logical_operator，会调用函数RC rc = create_logical_plan(sql_event, logical_operator);，函数中第二个参数为传出类参数，也即需要生成的逻辑算子。 函数会调用RC LogicalPlanGenerator::create(Stmt *stmt, unique_ptr<LogicalOperator> &logical_operator)，其中会根据 Stmt 的类型来调用create_plan函数。
