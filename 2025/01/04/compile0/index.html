编译原理八股文——绪论

绪论

什么是编译程序

翻译程序和解释程序的区别：

  • 翻译程序：将源代码翻译成目标代码
  • 解释程序：将目标代码直接得出结果，目标代码只是其中间过程
  • 编译程序将全部源代码先成功编译后才会运行，例如编译 C++ 程序，如果遇到错误一条语句都不会运行
  • 解释程序会将源代码一行一行的翻译成机器语言并直接运行，例如运行 python 程序、bash 脚本，程序会一直执行到发生错误
  • 编译程序的错误报告是在整个程序编译完成后，解释程序的错误报告是在对应报错的指令后

编译程序和汇编程序的区别：

  • 编译程序：将高级语言翻译成低级语言
  • 汇编程序：将汇编语言翻译成机器二进制码

编译程序的组成结构

词法分析 Scanner

任务：根据语言的构词规则识别单词符号

从左向右逐行扫描源程序的字符，识别出各个单词，确定单词的类型，将识别出的单词转换成统一的机内表示 —— token

语法分析 Parser

任务：根据语言的语法规则对词法分析的输出 token 串进行分析，识别出一个一个语法成分，并进行语法检查

在 token 序列中识别出各类短语，并构造语法分析树

语义分析 Semantic Analyzer

任务：根据语言的语义规则对语法分析所识别的各种语法成分的意义(即语义)进行确定并加以处理

中间码生成器 Intermediate Code Generator

任务：从源程序的树形或其它形式，产生源程序的中间代码

包括三地址码和语法树，三地址码用三元式/四元式表示

目标码生成器 Target Code Generator

任务：将中间代码转换成汇编代码或者机器语言

目标代码生成的一个重要任务是为程序中使用的变量合理分配寄存器

代码优化

任务：通过等价程序变换改进代码，使其与逆行的更快有一些、占用空间更小一些

符号表管理

符号表管理是一个贯穿编译全过程的工作，编译程序在分析源程序时，需要记录标识符的各种属性信息

在语义分析和代码生成阶段，还要对建立的符号表进行检索，提取相应的属性信息 —— 类型、作用域、分配存储信息

错误处理 Error Handler

  • 错误可发生在编译的各个阶段，错误处理也是贯穿编译全过程
  • 词法分析阶段可查出的错误，如标识符的组成不符合词法规则
  • 语句结构错误是在语法分析中可查出的错误
  • 语义分析阶段可查出的错误，即结构正确，但所涉及的操作无意义或错误
  • 在编译时查出的，Compile-time error，在运行时表现才表现出来的错误叫 Run-time error

编译程序的前端和后端

  • 前端：分析部分，和源语言相关
  • 后端：综合部分，和目标语言相关

前端包括编译逻辑结构中的分析部分即词法分析、语法分析、语义分析和中间代码生成，除此还包括符号表建造及相应分析中的错误处理以及与机器无关的优化部分

后端包括与目标机有关的部分，即综合部分，它包括目标代码生成及生成期间对符号表的相应检索操作和错误处理操作，以及与机器相关的代码优化部分

将编译系统划分为前后端，有利于移植编译系统和利用后端为同一目标机配置不同语言的编译系统

编译程序中的 遍 pass

对源程序(或其中间形式)从头至尾扫描一次并进行有关加工处理，生成新的中间形式或最终目标程序，称为一遍

  • 目标质量高低，高则多遍
  • 机器内存大小，小则多遍
  • 源语言简繁：繁则多遍
  • 设计人员多少：多则多遍
